94. Binary Tree Inorder Traversal

Given the root of a binary tree, return the inorder traversal of its nodes’ values.

Example 1:

Input: root = [1,null,2,3]
Output: [1,3,2]

Example 2:

Input: root = []
Output: []

Example 3:

Input: root = [1]
Output: [1]

Constraints:

  • The number of nodes in the tree is in the range [0, 100].

  • -100 <= Node.val <= 100

Solution

Runtime: 3 ms, faster than 49.93% of C online submissions for Binary Tree Inorder Traversal.

Memory Usage: 6 MB, less than 51.56% of C online submissions for Binary Tree Inorder Traversal.

Discussion:

写这道题的时候发现我的数据结构知识有点要还给我的老师了，什么叫做inorder都忘记了，所以说，还是得老老实实看书233333，这道题使用了迭代的思路来完成。当该指点没有左支点的时候，就推出，然后把这个支点的数据写进入。

然后访问右支点的时候如果有数据就再次进入迭代，直到该节点既没有左支点又没有右支点，就写入数据。

这个地方实际上非常好理解，因此不做过多的解释。问题是最后的地方，在调用函数的时候首先就会进入迭代完成函数的调用，然后再进行结果的处理，首先分配函数空间，然后复制数组，将返回的size传给指针，最后返回函数。这里需要注意一下.

一个点：memcpy为什么不能完整的复制数组呢? 我在VS2019上运行了一下，然后发现了问题所在，好家伙前面数组用错了2333333，改完之后啥问题都没有了，真好，我就是个傻子。
